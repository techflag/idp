# TechFlag IDP Trademark Guidelines

These guidelines explain how to refer to TechFlag, TechFlag IDP, and
旗讯OCR智能文档处理平台 when using, modifying, or redistributing the open-source
community edition.

They are intended to keep the community edition easy to fork while protecting
users from confusion about what is an official TechFlag release.

## Protected Names

The following names identify software, documentation, workflows, user
interfaces, and releases created or maintained by TechFlag:

- TechFlag
- TechFlag IDP
- TechFlag IDP Community
- 旗讯OCR智能文档处理平台
- 旗讯 IDP

Project names, product names, logos, icons, domain names, screenshots, release
badges, and similar brand identifiers may also indicate TechFlag origin or
endorsement depending on context.

## What The License Covers

The source code is licensed under the GNU Affero General Public License v3.0
only (`AGPL-3.0-only`), unless a file or third-party notice states otherwise.

That license grants rights to use, study, modify, run, and redistribute the
covered software under its terms. It does not grant a separate right to use
TechFlag names, logos, product names, or branding in a way that suggests
official endorsement, official operation, or an official release.

## Allowed Uses

You may use TechFlag names in truthful, non-confusing ways, for example:

- To say that your project is based on TechFlag IDP Community.
- To link back to the original repository.
- To state compatibility, such as "works with TechFlag IDP Community".
- To preserve copyright, license, NOTICE, AUTHORS, and source attribution
  notices.
- To discuss the project in articles, videos, issue reports, benchmarks, or
  technical comparisons.

These uses should be accurate and should not imply that TechFlag endorses,
hosts, certifies, maintains, or supports your modified version.

## Modified Versions And Forks

If you distribute a modified version, use a name that is clearly distinguishable
from the official project.

Recommended wording:

- "Fork of TechFlag IDP Community"
- "Based on TechFlag IDP Community"
- "Modified build of TechFlag IDP Community"

Avoid names that could be confused with an official release, such as:

- "TechFlag IDP Pro" without permission
- "Official TechFlag IDP"
- "TechFlag Cloud"
- Any name that suggests TechFlag operates or endorses the modified service

If your fork is public-facing, make the fork status clear in the README, UI
footer, release notes, or about page.

## Not Allowed Without Permission

Please do not use TechFlag names, logos, or branding to:

- Claim or imply official TechFlag endorsement.
- Present a modified build as an official TechFlag release.
- Operate a public service that appears to be run by TechFlag.
- Register confusingly similar domain names, social accounts, package names, or
  marketplace listings.
- Use TechFlag branding in advertising or sales materials in a way that causes
  confusion about source, sponsorship, or support.
- Remove or alter attribution in a way that makes the origin of the project
  misleading.

## Logos, Icons, And Screenshots

Screenshots may be used for documentation, commentary, reviews, tutorials, or
issue reports when the context is clear.

Do not use TechFlag logos or visual identity as the primary branding for a
modified product or hosted service unless you have written permission.

If you modify the UI, remove or clearly qualify branding that would make users
believe they are using an official TechFlag-operated product.

## Packages And Registries

When publishing packages, containers, demos, templates, or hosted builds based
on this project, choose names that clearly identify your distribution as your
own.

Good examples:

- `your-org-idp`
- `your-org-techflag-idp-fork`
- `idp-community-custom`

Avoid package names that make the build look official unless you are publishing
on behalf of TechFlag.

## Reporting Confusing Use

If you see a project or service that appears to misuse TechFlag branding or
creates confusion about official releases, please contact the project
maintainers through the repository issue tracker or the contact channel listed
by TechFlag.

## Relationship To Other Files

These guidelines do not replace:

- `LICENSE`, which contains the AGPL-3.0-only license text.
- `NOTICE`, which explains attribution and project identity.
- `AUTHORS`, which identifies project authorship and contribution intent.

If there is a conflict between these guidelines and a written agreement with
TechFlag, the written agreement controls.
