# TechFlag IDP Community v20260621

TechFlag IDP Community is now available as a public open-source snapshot.

This first release focuses on a local-first document AI workbench: upload real files, parse them with OCR, inspect document trees and evidence, run structured extraction, and turn repeatable extraction logic into basic reusable workflows.

## Why This Release Matters

Most document extraction demos hide the hard parts: OCR quality, table structure, evidence selection, provider configuration, and reproducible workflows. TechFlag IDP Community keeps those parts visible so developers can inspect, debug, and extend the full document AI loop.

## Highlights

- **Local-first startup**: SQLite and local object storage by default.
- **MinerU by default**: MinerU is the default OCR and document parsing engine.
- **Bring your own LLM**: OpenAI-compatible provider configuration for real AI extraction.
- **Evidence-grounded extraction**: extract from located document evidence instead of blindly sending the whole file to the model.
- **Document review workbench**: inspect pages, OCR blocks, tables, document trees, and extraction results.
- **Temporary extraction**: describe what you want to extract; the system locates and extracts once.
- **Basic document applications**: save repeatable extraction steps into lightweight workflows.
- **English and Chinese UI**: browser language detection plus in-app switching.
- **Public release guardrails**: export checks block secrets, local runtime files, databases, and private artifacts.

## Quick Start

Backend:

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -r requirements.txt
cp .env.local.example .env.local
alembic -c alembic.ini upgrade head
python scripts/diagnose_auth.py --ensure-admin --password demo-pass
./start.sh
```

Frontend:

```bash
cd frontend
npm ci
npm run dev -- --host 0.0.0.0
```

Open:

```text
http://127.0.0.1:5173/idp/
```

Default local admin:

```text
Username: idp-admin
Password: demo-pass
```

## Provider Notes

The system can start without provider keys, but real parsing and extraction need provider configuration.

- Apply for MinerU: `https://mineru.net/?source=github`
- Configure OCR parsing: `MINERU_TOKEN=your-mineru-token`
- Configure LLM extraction with an OpenAI-compatible provider:

```bash
DASHSCOPE_API_KEY=your-llm-key
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=qwen3.6-27b
```

MinerU cloud must be able to fetch uploaded file URLs. For real parsing, configure OSS or expose the backend through a public URL with `BACKEND_PUBLIC_BASE_URL`.
