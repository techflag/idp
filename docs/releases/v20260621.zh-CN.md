# TechFlag IDP 社区版 v20260621

TechFlag IDP 社区版现在作为公开开源快照发布。

这是第一个社区版 release，核心是一套本地优先的文档 AI 工作台：上传真实文件，通过 OCR 解析，查看文档树和证据，运行结构化抽取，并把可重复的抽取逻辑沉淀为基础可复用工作流。

## 为什么发布这个版本

很多文档抽取 demo 会隐藏真正困难的部分：OCR 质量、表格结构、证据选择、provider 配置、可复用流程。TechFlag IDP 社区版把这些环节展示出来，让开发者可以检查、调试和扩展完整的文档 AI 链路。

## 重点能力

- **本地优先启动**：默认 SQLite 和本地对象存储。
- **默认 MinerU**：MinerU 是默认 OCR 和文档解析引擎。
- **自带 LLM 接入方式**：通过 OpenAI-compatible provider 配置真实 AI 抽取。
- **证据化抽取**：从定位后的文档证据中抽取，而不是把整份文件盲目交给模型。
- **文档审阅工作台**：查看页面、OCR 块、表格、文档树和抽取结果。
- **临时抽取**：描述想提取的数据，系统定位并执行一次性抽取。
- **基础文档应用**：将可重复抽取步骤保存为轻量工作流。
- **中英文 UI**：支持浏览器语言识别和页面内切换。
- **公开发布护栏**：导出检查会阻断密钥、本地运行文件、数据库和私有产物。

## 快速启动

后端：

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -r requirements.txt
cp .env.local.example .env.local
alembic -c alembic.ini upgrade head
python scripts/diagnose_auth.py --ensure-admin --password demo-pass
./start.sh
```

前端：

```bash
cd frontend
npm ci
npm run dev -- --host 0.0.0.0
```

浏览器打开：

```text
http://127.0.0.1:5173/idp/
```

默认本地管理员账号：

```text
账号：idp-admin
密码：demo-pass
```

## Provider 说明

系统可以不配置 provider key 启动，但真实解析和抽取需要配置 provider。

- MinerU 申请地址：`https://mineru.net/?source=github`
- OCR 解析配置：`MINERU_TOKEN=你的 MinerU Token`
- LLM 抽取配置 OpenAI-compatible provider：

```bash
DASHSCOPE_API_KEY=你的 LLM Key
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=qwen3.6-27b
```

MinerU 云端必须能访问上传文件 URL。真实解析需要配置 OSS，或通过 `BACKEND_PUBLIC_BASE_URL` 暴露后端公网地址。
